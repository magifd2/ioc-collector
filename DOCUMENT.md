# プロジェクト引き継ぎ書: ioc_collector

## 1. プロジェクト概要

### 1.1. プロジェクト名
`ioc_collector`

### 1.2. 目的
セキュリティ関連のニュース記事やインシデントレポート（URL、テキスト、CVE-ID等）を起点とし、関連情報をWebから自律的に収集・分析するCLIツールです。最終的に、インシデントの概要をまとめたMarkdownレポートと、IoC（Indicators of Compromise）情報を構造化したSTIX 2.1形式のJSONファイルを生成することを目的とします。

### 1.3. コア技術
- **言語:** Python 3.11+
- **メインエンジン:** Google Gemini Pro (Vertex AIのGoogle Search Grounding機能を利用)
- **パッケージ管理:** uv
- **CLIフレームワーク:** Typer
- **データ構造化:** Pydantic, STIX2

---

## 2. 詳細仕様

### 2.1. 主要機能
1.  **入力受付**:
    -   **入力形式:** コマンドライン引数 (`-t`, `--target`)、標準入力、ファイル入力 (`-f`, `--file`) に対応。
    -   **入力内容:** URL、複数行の自然言語テキスト、CVE-IDなど。

2.  **調査対象の確認 (インタラクティブモード)**:
    -   LLMが解釈した調査対象のインシデント概要を提示し、実行前にユーザーの確認を求めます。
    -   `--non-interactive` フラグで確認をスキップ可能です。

3.  **情報収集 (Web Research)**:
    -   **エンジン:** Gemini Pro
    -   **対象:** マルチリンガルでWeb全体から情報を収集。

4.  **情報分析・構造化**:
    -   収集したテキストから、インシデント概要とIoC情報を抽出・整理します。
    -   **対象IoC:** IPアドレス, URL, ドメイン名, ファイルハッシュ, プロセス名, ファイル名、その他検知可能な痕跡。

5.  **出力**:
    -   **ファイル名:** `(インシデント名)_(yyyymmdd_hhmmss)` の形式で統一。
    -   **Markdownレポート (`.md`):** インシデント概要、タイムライン、影響範囲、対策、IoC一覧、参考URLリスト。
    -   **構造化データ (`.json`):** STIX 2.1 形式。`Indicator`, `Malware`, `Report` 等のオブジェクトを使用。

### 2.2. CLIインターフェース案
```bash
# URLを直接指定
uv run ioc_collector --target "https://example.com/news/article.html"

# 自然言語で指定
uv run ioc_collector --target "2023年に発生した〇〇社のランサムウェアインシデント"

# ファイルから入力
uv run ioc_collector --file "incident_query.txt"

# パイプで渡す
cat "incident_query.txt" | uv run ioc_collector

# 出力ディレクトリを指定
uv run ioc_collector -t "..." -o ./my_reports

# 確認なしで実行
uv run ioc_collector -t "..." --non-interactive
```

### 5.2. 課題: テストコードにおける `SyntaxError`
`typer.testing.CliRunner` を使用したテスト (`tests/test_main.py`) の実装中に、CLIへの `input` をシミュレートする部分で解決困難な `SyntaxError` が発生しました。

- **現象:**
  `runner.invoke(app, ..., input="y
")` のように改行を含む入力をテストコードに記述すると、`write_file` ツールがこのバックスラッシュを正しくエスケープできず、結果として不正なPythonコードがファイルに書き込まれ、`pytest` のテスト収集フェーズで `SyntaxError: unterminated string literal` が発生します。

- **試したこと:**
  - `
`, `r"y
"`, `
` など、様々なエスケープ方法を試しましたが、`write_file` ツールの内部的な文字列処理により、意図した通りのコード生成ができませんでした。

- **引き継ぎ先への依頼事項:**
  - この問題のため、`tests/test_main.py` 内のインタラクティブモードに関するテストは、一時的に削除されています。
  - **開発を再開する際は、まず `typer.confirm` を含むインタラクティブな処理をテストする方法を確立してください。** `CliRunner` の `input` 引数の挙動を再確認するか、または別のテストアプローチ（`pexpect` の利用など）を検討する必要があるかもしれません。
  - この問題が解決すれば、`main.py` の実装は概ねテストがパスする状態に近づくはずです。まずは `tests/test_main.py` を完全にパスさせること（Green）を目標としてください。
